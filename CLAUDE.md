# DiagramCore — контекст для агентів

Цей файл — точка входу для будь-якого агента, що працює в цьому репозиторії.

## Що це за проєкт

DiagramCore: YAML-формат (`*.dc.yaml`) для опису архітектурних діаграм +
CLI (`dc`) для валідації, генерації AI-контексту та рендерингу (D2/Mermaid/SVG)
+ веб-редактор (`web/`, React Flow). Концепт — `docs/concept.md`.
Специфікація формату — `docs/format.md`.

## Де що шукати

- **`PLAN3.md`** — АКТИВНИЙ план (фаза 11: функціональне ядро редактора)
  з Acceptance Criteria. Виконується строго послідовно, крок за кроком.
- **`docs/plans/`** — виконані плани: `PLAN.md` (фази 0–9, там же
  «Загальні рішення»), `PLAN2.md` (фаза 10, UI/UX),
  `progress-log-phases-0-10.md` (журнал тих фаз).
- **`docs/progress-log.md`** — журнал поточної фази: що зроблено, коли,
  яким комітом. Перевіряти перед стартом роботи, щоб знати поточний стан.
- **`docs/deviations.md`** — відхилення від зафіксованого плану: коли
  довелось відійти від плану (змінилося API залежності, рішення виявилось
  нездійсненним тощо) і яке рішення прийнято натомість.
- **`docs/bugfixes.md`** — журнал виправлених багів (симптом, причина,
  файли, коміт), окремо від планових кроків фаз.

## Правила виконання (повний текст — розділ «Правила для агента-валідатора» в PLAN3.md)

1. Крок вважається завершеним лише коли ВСІ його AC проходять — перевіряти буквально.
2. Перед закриттям кроку — регресія: `go test ./...`, `go vet ./...`,
   `./dc validate examples/*.dc.yaml`; у `web/` — `npm test` + `npm run build`;
   якщо крок зачіпає Go/WASM — ще `make wasm && make wasm-test`.
3. Після кожного кроку — commit `phase11-step<M>: <опис>`, і додати запис
   у `docs/progress-log.md`.
4. Якщо доводиться відхилитись від плану — не мовчки: записати в
   `docs/deviations.md` (крок, причина, прийняте рішення) і продовжити.
5. Зафіксовані рішення (розділ «Загальні рішення» в `docs/plans/PLAN.md`
   та «Рішення фази» в PLAN3.md) не переглядати без згоди власника.

## Загальні рішення (не переглядати без згоди власника)

Повний список — «Загальні рішення» у `docs/plans/PLAN.md` + «Рішення фази»
у PLAN3.md. Коротко: YAML-ядро — джерело правди, канва (React Flow) —
проєкція; Go CLI; D2-рендеринг; layout (позиції, розміри, інстанс-стилі)
окремим файлом `<name>.layout.json`; стилі рівня типу — в YAML
`custom_types`; вкладеність — `parent:` на вузлі; flow тільки по існуючих
ребрах; web-редактор — TypeScript + Go→WASM валідатор.
