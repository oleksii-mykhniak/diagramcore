# DiagramCore — контекст для агентів

Цей файл — точка входу для будь-якого агента, що працює в цьому репозиторії.

## Що це за проєкт

DiagramCore: YAML-формат (`*.dc.yaml`) для опису архітектурних діаграм +
CLI (`dc`) для валідації, генерації AI-контексту та рендерингу (D2/Mermaid/SVG).
Концепт — `docs/concept.md`. Специфікація формату — `docs/format.md`.

## Де що шукати

- **`PLAN.md`** — повний покроковий план реалізації з Acceptance Criteria.
  Виконується строго послідовно, фаза за фазою.
- **`docs/progress-log.md`** — журнал виконаних кроків: що зроблено, коли,
  яким комітом. Перевіряти перед стартом роботи, щоб знати поточний стан.
- **`docs/deviations.md`** — відхилення від зафіксованого плану ("інсайди"):
  коли довелось відійти від PLAN.md (змінилося API залежності, рішення
  виявилось нездійсненним тощо) і яке рішення прийнято натомість.

## Правила виконання (повний текст — розділ "Правила для агента-валідатора" в PLAN.md)

1. Крок вважається завершеним лише коли ВСІ його AC проходять — перевіряти буквально.
2. Перед закриттям кроку — регресія: `go test ./...`, `go vet ./...`,
   `./dc validate examples/*.dc.yaml` (з фази 1+); у `web/` — `npm test` + `npm run build`.
3. Після кожного кроку — commit `phase<N>-step<M>: <опис>`, і додати запис
   у `docs/progress-log.md`.
4. Якщо доводиться відхилитись від плану — не мовчки: записати в
   `docs/deviations.md` (крок, причина, прийняте рішення) і продовжити.
5. Зафіксовані рішення (розділ "Загальні рішення" в PLAN.md — формат YAML,
   мова CLI Go, розділення layout тощо) не переглядати без згоди власника.

## Загальні рішення (не переглядати без згоди власника)

Див. розділ "Загальні рішення" у `PLAN.md`. Коротко: YAML-ядро, Go CLI,
D2-рендеринг, layout окремим файлом, flow тільки по існуючих ребрах,
web-редактор — TypeScript + Go→WASM валідатор.
